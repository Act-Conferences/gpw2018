Workshop: Perl on embedded DevicesWorkshop: Perl on embedded devices

Das Ziel dieses Workshops ist es, den Teilnehmern den sicheren Umgang von Softwarenetwicklung und -bereitstellung auf IoT-Geräten zu vermitteln. Außerdem soll von vornherein auf Nicht-Funktionale Anforderungen wie Security-Updates eingegangen werden. As goal of this workshop the participants learn the safe and reliable way to develop and deploy software on IoT devices. Being able to fulfill non-functional requirements as security updates is conceptionally included.

§9: Wat sull dä Quatsch? §9: Wat sull dä Quatsch? (What is it good for?)

  1. Man kann Erfahrungen mit dem Entwickeln von Software für Geräte sammeln, auf die man im Zweifel keinen Zugriff hat, wenn ein Supportfall auftritt. One can make experiences with developing of software for devices without having access to the device in case of support conditions.
  2. Es werden Konzepte vermittelt, wie man SaaS mittels embedded Devices vertreiben kann und u.U. sogar Abrechnungsmodelle. We'll talk about concepts using embedded devices for SaaS models, even partially for accounting.
  3. Es können Experimente mit verschiedenen Betriebskonzepten gemacht und nachvollzogen werden (z.B. Debian auf ext4 vs. Poky auf Squashfs). Participants can do experiments with different operation concepts (e.g. Debian using ext4 vs. Poky on squashfs).
  4. Programmierer können bisher unbekannte Probleme erfahren und proaktive bessere Softwarequalität herstellen. Developers could experience previously unknown problems and thus may result in proactive improved software quality.
  5. Maintainer können erleben, was Entscheidungen für Konsequenzen erzeugen. Maintainers could experience which decisions produces which consequences.

Der Workshop behandelt folgende Themenbereiche The workshop covers following topics

  1. Grundlage von Images für embedded Systeme Basis of images for embedded systems
  2. Kennenlernen der Infrastruktur Becoming familiar with the infrastructure
  3. Einloggen auf dem Buildsystem für den Workshop Getting logged in on the build machine for the workshop
  4. Hinzufügen einer einfachen Perl-Anwendung Adding a simple Perl application
  5. Erstellen eines neuen Images mit der Anwendung Building a new image with that application
  6. Flashen des Demo-Boards flash the built image on the demo board
  7. Test der Anwendung Testing the application
  8. Hinzufügen eines neuen Modules Adding a new perl module
  9. Verwendung des Modules in der Test-Anwendung Using the newly added Perl module in the test application
  10. Flashen und Testen Flash and test the updated test application
  11. eigene Schritte individual steps

selbständiges Arbeiten autonomous work

Ab hier können die Teilnehmer verschiedene eigene Schritte ausprobieren - vom Hinzufügen einer eigenen, existierenden Anwendung über Experimente mit elektronischen Bauteilen bis hin zum Testen der eigenen Perl-Module - alles ist möglich. Starting from here each perticipant can do several individual steps - from adding own, existing applications over experimenting with electronic components up to testing and adopting maintained Perl modules.

Bei Interesse kann der individuelle Abschnitt im Rahmen eines Hackathons am Samstag fortgesetzt werden. On request the individual part can continued with in the hackathon sessions organized on Saturday.
